### 步骤1：检查当前环境状态（2分钟）
```
在微信开发者工具中：
1. 打开项目
2. 点击"云开发"按钮
3. 查看当前环境ID
4. 确认是否是：tengfei-workspace-7ef9ma8b7670ea

### 步骤2：如果不是目标环境（3分钟）
```
情况A：环境列表中有目标环境
1. 点击环境下拉菜单
2. 选择"tengfei-workspace-7ef9ma8b7670ea"

### 步骤3：上传翻译云函数（3分钟）
```
1. 在"云开发"面板点击"云函数"
2. 点击"新建云函数"
3. 函数名称输入：translate
4. 选择本地文件夹：cloud-functions/translate
5. 点击"上传并部署"
6. 等待部署完成（约1-2分钟）
```

### 步骤4：配置环境变量（2分钟）
```
1. 在"云开发"面板点击"环境设置"
2. 找到"环境变量"配置
3. 点击"添加环境变量"
4. 添加以下3个变量：

变量名：TENCENT_SECRET_ID
变量值：[你的SecretId]

变量名：TENCENT_SECRET_KEY  
变量值：[你的SecretKey]

变量名：TENCENT_REGION
变量值：ap-shanghai

5. 点击"确定"保存
```

### 步骤5：立即测试（2分钟）
```
1. 重新编译小程序
2. 访问翻译页面：pages/translate/translate
3. 输入"hello"，点击翻译
4. 观察结果：

✅ 成功情况：
   显示"腾讯云翻译" + 正确翻译"你好"

⚠️ 警告情况：
   显示"本地词典" + 翻译结果

```

## 🔧 如果遇到问题

### 问题1：环境ID不匹配
```
当前情况：微信开发者工具中的环境ID不是目标ID
临时解决方案：
1. 使用当前环境ID继续部署
2. 更新小程序代码中的环境ID
3. 修改 client-mini-wechat/app.js 中的 env 配置
```

### 问题2：云函数部署失败
```
查看日志：
1. 在云函数列表中找到"translate"
2. 点击"详情"
3. 查看"日志"标签页
4. 根据错误信息解决

常见错误：
- 依赖安装失败 → 检查package.json
- 代码语法错误 → 检查index.js
- 权限不足 → 检查环境权限
```

### 问题3：环境变量不生效
```
验证方法：
1. 创建测试云函数
2. 在代码中打印环境变量
3. 调用测试函数查看输出

快速测试代码：
exports.main = async () => {
  console.log('环境变量测试:');
  console.log('SECRET_ID:', process.env.TENCENT_SECRET_ID);
  return process.env.TENCENT_SECRET_ID;
};
```
