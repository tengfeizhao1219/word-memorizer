## 1. 项目概述

### 1.1 项目背景

学习英语过程中，记录和复习生词是刚需。现有产品要么功能复杂，要么需要付费，缺少一款轻量、免费、多端同步的生词本应用。

### 1.2 产品定位

| 维度 | 描述 |
|------|------|
| 目标用户 | 英语学习者（学生、职场人士） |
| 核心场景 | 阅读/看剧时遇到生词，快速记录并复习 |
| 差异化 | 轻量、免费、多端实时同步 |
| 平台 | 微信小程序 + Web 端 |

### 1.3 项目目标

| 阶段 | 目标 | 时间 |
|------|------|------|
| MVP | 核心功能可用，可上线测试 | 3 周 |
| V1.0 | 功能完善，正式发布的版本 | 6 周 |
| V2.0 | 加入 AI 功能（OCR、智能释义） | 3 个月 |

### 1.4 成功指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 注册用户 | 1000+ | 上线 3 个月 |
| 日活 (DAU) | 100+ | 上线 3 个月 |
| 生词记录数 | 10 万+ | 累计 |
| 用户留存 (7 日) | 30%+ | 健康线 |

### 2.2 用户故事

```
作为 英语学习者
我想要 快速记录生词
以便于 后续复习和掌握

作为 多设备用户
我想要 数据在小程序和 Web 端同步
以便于 随时随地学习

作为 备考学生
我想要 按分类管理生词
以便于 针对性复习
```

### 2.3 用户流程

#### 登录流程
```
用户打开小程序 → 点击登录 → 微信授权 → 获取用户信息 → 创建/更新用户记录 → 进入首页
```

#### 添加生词流程
```
点击添加按钮 → 输入单词 → (可选) 输入释义/例句 → 选择分类 → 保存 → 同步到云端
```

#### 复习流程
```
进入复习页面 → 查看待复习单词 → 显示/隐藏释义 → 标记掌握程度 → 更新下次复习时间
```
