## 🔧 **技术突破总结**

### **自动化部署方法**
```bash
# 关键命令
npx tcb fn deploy [函数名] -e [环境ID] --yes

# 关键参数
--yes          # 跳过所有交互确认
--force        # 覆盖已存在的函数
```

### **避免付费弹窗的技巧**
```
内存配置: 使用256MB (不会触发付费套餐)
超时配置: 5-30秒 (根据函数复杂度)
运行时: Nodejs16.13 或 Nodejs18.15
```

### **配置文件标准化**
```json
// tcb.json 配置文件
{
  "functions": [{
    "name": "函数名",
    "timeout": 10,
    "runtime": "Nodejs16.13",
    "handler": "index.main"
  }]
}
```

## ⚠️ **常见问题解决**

### **问题1: 环境无效 (INVALID_ENV)**
```
原因: HTTP访问服务未配置
解决: 为每个云函数创建HTTP访问服务
```

### **问题2: 数据库连接失败**
```
原因: 数据库集合未创建或权限错误
解决: 创建5个集合并设置正确权限
```

### **问题3: 小程序编译错误**
```
原因: 依赖未安装或配置错误
解决: 重新构建npm，检查配置文件
```

### **问题4: API返回500错误**
```
原因: 云函数代码错误
解决: 查看云函数日志，修复代码
```
